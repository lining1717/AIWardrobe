# AIWardrobe · UI 重设计需求文档

> 本文档供 Figma 设计参考，描述 AIWardrobe 应用的完整功能、页面结构与设计方向。

---

## 一、产品概述

**AIWardrobe（AI 智能衣橱）** 是一款结合 AI 视觉识别 + 天气感知 + 个性化推荐的智能穿搭管理应用。

**核心价值**：
- 📸 拍照即录入——AI 自动去背景 + 识别衣物属性
- 🌤️ 看天穿衣——结合实时天气智能推荐穿搭
- ✨ AI 搭配师——根据目标场景/星座生成穿搭建议
- 👗 虚拟试穿——上传人物图预览上身效果

**目标用户**：注重穿搭品质、希望系统化管理衣橱的都市人群（偏女性用户）。

---

## 二、核心使用场景

| # | 场景 | 用户旅程 |
|:---|:---|:---|
| 1 | 录入衣物 | 拍照/相册选图 → AI 去背景 → AI 识别属性 → 编辑确认 → 入库 |
| 2 | 浏览衣橱 | 按分类浏览 → 搜索/筛选 → 查看详情 → 删除 |
| 3 | AI 推荐 | 设定场景目标 → AI 结合天气+星座+衣橱生成推荐 → 查看搭配理由 |
| 4 | 手动搭配 | 从衣橱自选上装/下装/鞋 → 实时预览组合 |
| 5 | 虚拟试穿 | 衣物详情页 → 上传人物图 → AI 生成试穿效果图 |

## 四、页面详情

### 📄 页面 1：首页 Home（仪表盘）

**定位**：打开 App 第一眼，聚合当日关键信息

**内容区块**：
1. **顶部 Header**
   - 今日日期标签
   - 右上角：刷新按钮 + 设置按钮

2. **天气卡片**
   - 温度大字（主视觉）
   - 位置 + 天气状况
   - 体感温度、湿度、风力（次级信息）

3. **衣橱轮播卡片**
   - 横向滑动展示衣物单品
   - 每张卡：图片 + 名称 + 分类标签 + 描述
   - 左右箭头切换 + 底部圆点指示器
   - 「查看全部」按钮 → 跳衣橱页
   - 空衣橱时引导去录入页

4. **星座运势卡片**
   - 星座名徽章
   - 运势概述
   - 三宫格：心情 / 幸运色 / 幸运数字
   - 当日建议
   - 未设置星座时显示「设置星座」引导

**桌面布局**：12 列网格，天气(5) + 衣橱(7) + 星座(12)

---

### 📄 页面 2：录入 Entry

**定位**：添加新衣物的入口

**两个视图状态**：

**状态 A：上传视图**
- 标题「录入」+ 设置按钮
- Hero 引导卡（带跳转衣橱/推荐按钮）
- 01/02/03 三步引导说明
- Upload 上传组件

**状态 B：编辑视图**（上传成功后切换）
- 顶部：返回 + 标题 + 保存按钮
- 正方形图片预览（去背景后的图）
- 基本信息卡：名称输入 + 分类下拉
- 特征卡：颜色、风格（逗号分隔）、季节、用途
- 描述文本域

**Upload 组件能力**：
- 相册选择 / 相机拍照 / 拖拽上传 / 批量上传
- 上传进度条（去背中 → 分析中 → 完成）
- 移动端调系统相机，PC 端调 webcam

---

### 📄 页面 3：衣橱 Wardrobe

**定位**：浏览、搜索、管理所有衣物

**内容区块**：
1. **顶部 sticky 玻璃 header**（内嵌 FilterBar）
   - 搜索框（关键词）
   - 季节筛选胶囊（春/夏/秋/冬）
   - 风格筛选胶囊（休闲/正式/运动/商务/复古/极简/日常/通勤）

2. **四个分类分区**
   - 👕 上装（tops）
   - 👖 下装（bottoms）
   - 👟 鞋（shoes）
   - 💍 配饰（accessories）
   - 每个分区：分类标题 + 数量徽章 + 卡片网格

3. **衣物卡片**
   - 图片 + 名称 + 删除按钮
   - 点击跳转详情页
   - 删除二次确认

4. **空状态**
   - 📦 图标 + 无匹配提示

**响应式网格**：1 列(移动) / 2 列 / 3 列 / 4 列(桌面)

---

### 📄 页面 4：衣物详情 ClothesDetail

**定位**：查看单品完整信息 + AI 虚拟试穿

**内容区块**：
1. **顶部**：返回 + 标题
2. **左侧图片卡**：大图 + 名称 + 分类
3. **右侧属性卡**：
   - 描述
   - 颜色
   - 风格标签（多个）
   - 季节标签（多个）
   - 用途标签（多个）
4. **底部试穿区**（全宽）：
   - 上传人物图按钮
   - 人物图预览
   - 「生成试穿」按钮
   - 试穿结果图展示

**桌面布局**：12 列网格，图片(5) + 属性(7) + 试穿(12)

---

### 📄 页面 5：穿搭 Outfit（手动搭配）

**定位**：用户自由组合搭配

**内容区块**：
1. **标题 + 随机洗牌按钮**
2. **季节过滤胶囊**（全部/春/夏/秋/冬）
3. **左侧大图区**：三层堆叠展示
   - 上装（44% 高度）
   - 下装（34% 高度）
   - 鞋（22% 高度）
4. **右侧三张分类卡**：
   - 每张：上/下页切换按钮 + 当前序号(n/总数) + 当前单品大图
   - 点击单品图跳详情页

**交互**：切换季节重新匹配 / 上下箭头切换单品 / 洗牌随机生成

---

### 📄 页面 6：AI 推荐 Recommendation

**定位**：AI 智能穿搭推荐（核心亮点）

**三个状态**：

**状态 A：初始态**
- Sparkles 图标 + 标题
- 模式选择三按钮：
  - balanced（平衡推荐）
  - goal_first（目标优先）
  - wardrobe_first（衣橱优先）
- 目标输入框 + 🎤 麦克风语音输入
- 生成按钮
- 当前定位显示

**状态 B：加载态**
- 双圈旋转 + 闪烁 Sparkles
- AI 思考文案

**状态 C：结果态**
1. **天气渐变大卡片**（蓝色渐变 + 巨型天气 emoji 模糊背景）
   - 温度、状况、体感、湿度、风力
2. **星座四宫格**
   - 星座 / 心情 / 幸运色 / 幸运数字
3. **目标回显** + **温度规则卡**
4. **AI 推荐文字**（Markdown 渲染 + 打字机光标效果）
5. **穿搭概要**
6. **三件套衣物卡**：上装 / 下装 / 鞋（图片+名称+描述+选择理由）
7. **配饰卡**：来自衣橱（绿徽章）/ 需购买（橙徽章）
8. **购买建议卡**：标题 / 理由 / 关键词 / 星座提示

**背景装饰**：两块模糊光晕（蓝/紫）

### 🛠 全局组件：设置 Settings

**定位**：应用配置中心（弹窗/抽屉）

**配置分区**：

   - 语言切换（中/英/日）

## 五、设计风格指引

> 基于目标用户偏好：**优雅、柔和、时尚、简约、温暖**

### 🎨 色彩建议

| 用途 | 建议色值 | 说明 |
|:---|:---|:---|
| 主背景 | `#FAF8F5` 暖米白 | 温暖不刺眼，替代冷白 |
| 卡片背景 | `#FFFFFF` 纯白 | 干净的卡片层级 |
| 主文字 | `#2D2A26` 暖墨黑 | 比纯黑柔和 |
| 次文字 | `#8B8378` 暖灰 | 信息层级 |
| 强调色 | `#C9A87C` 香槟金 / `#D4A5A5` 柔粉 | 优雅时尚，替代蓝色 |
| 辅助色 | `#A8B5A0` 鼠尾草绿 | 自然柔和 |
| 警示色 | `#D4928B` 陶土粉 | 柔和的提醒 |

**避免**：高饱和蓝、荧光色、冷灰、纯黑

### ✍️ 字体建议

- **标题**：衬线字体（如 Noto Serif SC / Playfair Display）—— 优雅感
- **正文**：无衬线（如 Noto Sans SC / Inter）—— 易读
- **数字**：可考虑轻量级无衬线 —— 现代感

### 📐 布局原则

1. **留白优先**：卡片间距充足，呼吸感强
2. **圆角统一**：建议 16-24px 大圆角，柔和
3. **柔和阴影**：避免硬阴影，用 `0 4px 20px rgba(0,0,0,0.06)`
4. **图片为主**：衣物图片是核心视觉元素，占据主要空间
5. **移动优先**：移动端单列，桌面端多列网格

### 🌸 视觉风格关键词

- 优雅（Elegant）
- 柔和（Soft）
- 时尚（Fashionable）
- 简约（Minimalist）
- 温暖（Warm）
- 女性友好（Female-friendly）

### ✨ 交互建议

- 过渡动画：300ms ease-out，柔和
- 卡片 hover：轻微上浮 + 阴影加深
- 加载状态：优雅的 skeleton / 柔和 spinner
- 页面切换：淡入淡出
- 推荐 AI 文字：打字机效果（保留）
- 轮播：自动 + 手动，过渡平滑

## 六、关键交互流程图

```
首页/录入 Tab → 上传图片 → [AI 去背景] → [AI 识别] → 编辑页 → 保存 → 入库

```

## 七、数据模型

### 衣物（ClothesItem）
| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | int | ID |
| category | enum | top / bottom / shoes / accessory |
| item | string | 衣物名称（如「白色棉质T恤」）|
| style_semantics | string[] | 风格标签（如 [休闲, 简约]）|

| description | string | 一句话描述 |
| image_url | string | 图片地址 |
| created_at | datetime | 录入时间 |

### 推荐结果（RecommendationResponse）
| 字段 | 类型 | 说明 |
|:---|:---|:---|
| weather | object | 天气信息 |
| horoscope | object | 星座运势 |
| temperature_rule | object | 温度规则 |
| recommendation_text | string | AI 推荐文字（Markdown）|
| outfit_summary | string | 穿搭概要 |
| suggested_top/bottom/shoes | object | 推荐单品 |
| suggested_accessories | object[] | 配饰建议 |
| purchase_suggestions | object[] | 购买建议 |
| selection_reasons | object | 选择理由 |

## 九、现有 UI 痛点（重设计时改进方向）

1. **视觉风格偏冷**：现有强调色为蓝色，与「温暖优雅」调性不符
2. **首页信息密度高**：天气+衣橱+星座堆叠，缺乏主次
3. **录入流程割裂**：上传与编辑分两屏，可考虑更流畅的一体化
4. **衣橱卡片信息少**：仅图片+名称，未展示风格/颜色等关键属性
5. **推荐结果信息量大**：需更好的视觉分层和阅读引导
6. **穿搭页较简陋**：三层堆叠效果可优化为更真实的搭配预览
